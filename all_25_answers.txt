# Easy & Simple Answers: JUnit, JaCoCo, Docker, Deployment, Puppet, Cloud (AWS & Azure)

1. What is JUnit and what are its features?

JUnit
JUnit is a Java testing framework used to write and run unit test cases.

Features of JUnit
- Helps in automated testing
- Supports unit testing
- Provides assert methods to check results
- Uses annotations like @Test
- Supports test suites
- Supports exception testing
- Easy integration with Maven, Gradle, Eclipse, IntelliJ
- Can generate test reports

2. What are the important annotations provided by JUnit?

Important JUnit 5 annotations
- @Test → marks a test method
- @BeforeEach → runs before every test
- @AfterEach → runs after every test
- @BeforeAll → runs once before all tests
- @AfterAll → runs once after all tests
- @Disabled → skips a test
- @DisplayName → gives custom test name
- @Nested → creates nested test classes
- @ParameterizedTest → runs test with multiple inputs

JUnit 4 annotations
- @Test
- @Before
- @After
- @BeforeClass
- @AfterClass
- @Ignore

3. What are the JUnit assert methods?

Assert methods are used to verify expected and actual results.

Common assert methods
- assertEquals(expected, actual) → checks if values are equal
- assertNotEquals(expected, actual) → checks if values are not equal
- assertTrue(condition) → checks if condition is true
- assertFalse(condition) → checks if condition is false
- assertNull(object) → checks if object is null
- assertNotNull(object) → checks if object is not null
- assertSame(expected, actual) → checks if both refer to same object
- assertNotSame(expected, actual) → checks if different objects
- assertArrayEquals(expected, actual) → compares arrays
- assertThrows() → checks if exception is thrown (JUnit 5)

4. What is the difference between JUnit 4 and JUnit 5?

JUnit 4
- Older version
- Uses @Before, @After
- Uses @BeforeClass, @AfterClass
- Uses @Ignore
- Limited features
- No built-in support for advanced extensions
- Less support for parameterized tests
- Monolithic

JUnit 5
- Newer and improved version
- Uses @BeforeEach, @AfterEach
- Uses @BeforeAll, @AfterAll
- Uses @Disabled
- More powerful and flexible
- Supports extension model
- Better parameterized test support
- Modular (Platform, Jupiter, Vintage)

Simple point
- JUnit 5 is more modern, flexible, and feature-rich than JUnit 4.

5. How do you test exceptions in JUnit 5?

In JUnit 5, exceptions are tested using assertThrows().

Example
import static org.junit.jupiter.api.Assertions.*;

@Test
void testException() {
    Exception ex = assertThrows(ArithmeticException.class, () -> {
        int result = 10 / 0;
    });
}

Simple explanation
- assertThrows() checks whether a specific exception occurs.
- If exception is thrown, test passes.

6. What is JaCoCo and why is it used?

JaCoCo
JaCoCo stands for Java Code Coverage.

It is a tool used to measure how much of your Java code is tested by JUnit or other tests.

Why it is used
- Checks code coverage
- Finds untested code
- Improves test quality
- Helps in CI/CD pipelines
- Generates HTML/XML/CSV reports

Simple point
- JaCoCo is used to know how much code is covered by tests.

7. How do you generate and run a JaCoCo test report?

JaCoCo is usually used with Maven or Gradle.

Using Maven
Add plugin in pom.xml:

<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.10</version>
    <executions>
        <execution>
            <goals>
                <goal>prepare-agent</goal>
            </goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals>
                <goal>report</goal>
            </goals>
        </execution>
    </executions>
</plugin>

Run commands
mvn clean test

Report location
- target/site/jacoco/index.html

Simple explanation
- Run tests
- JaCoCo tracks coverage
- It generates a report showing covered and uncovered code

8. How can you improve or increase code coverage using JaCoCo?

Ways to improve coverage
- Write more unit tests
- Test all methods
- Test if-else conditions
- Test loops
- Test exception cases
- Test boundary values
- Cover positive and negative scenarios
- Use parameterized tests

Simple point
- Increase coverage by testing all possible code paths and edge cases.

9. Define Docker and explain its key features.

Docker
Docker is a containerization platform used to build, package, and run applications in containers.

Key features
- Lightweight
- Fast startup
- Portable (runs anywhere)
- Consistent environment
- Easy deployment
- Isolation between applications
- Uses fewer resources than VMs
- Supports versioning through images

Simple point
- Docker packages an application and its dependencies into a container so it runs the same everywhere.

10. What are Docker images, Dockerfile, and Docker Hub?

Docker Image
A read-only template used to create containers.
It contains:
- Application code
- Libraries
- Dependencies
- Runtime

Dockerfile
A text file with instructions to build a Docker image.

Docker Hub
A cloud repository where Docker images are stored and shared.

Simple example
- Dockerfile = recipe
- Image = prepared package
- Container = running package
- Docker Hub = online image store

11. Describe the lifecycle of a Docker container.

Container lifecycle stages
1. Created → container is created
2. Running → container is actively running
3. Paused → temporarily stopped
4. Stopped/Exited → container is stopped
5. Restarted → started again
6. Removed → deleted permanently

Simple flow
- Create → Run → Stop/Pause → Restart → Remove

12. What are the key components of Docker architecture?

Key components
- Docker Client → user commands (docker run, docker build)
- Docker Daemon (dockerd) → runs in background and manages containers
- Docker Images → templates to create containers
- Docker Containers → running instances of images
- Docker Registry → stores images (example: Docker Hub)

Simple architecture
- Client → Daemon → Images/Containers → Registry

13. Differentiate between virtualization and containerization.

Virtualization
- Uses Virtual Machines (VMs)
- Each VM has its own OS
- Heavyweight
- Slower startup
- Uses more memory/CPU
- Better full OS isolation

Containerization
- Uses Containers
- Containers share host OS
- Lightweight
- Faster startup
- Uses fewer resources
- Process-level isolation

Simple point
- VM = full machine, Container = lightweight app environment

14. How do you start, stop, and kill a Docker container?

Commands
- Start container
docker start <container_id>

- Stop container
docker stop <container_id>

- Kill container (force stop)
docker kill <container_id>

Extra
- Run new container
docker run <image_name>

15. What is the purpose of the volume parameter in a Docker run command?

Volume parameter
The -v or --volume option is used to mount storage between:
- Host machine and container, or
- Docker managed volume and container

Purpose
- Store data permanently
- Prevent data loss when container is deleted
- Share files between host and container
- Useful for databases, logs, uploads

Example
docker run -v /host/data:/app/data myapp

Simple point
- Volumes are used for persistent data storage in Docker.

Different applications have different needs, so multiple deployment systems exist.

Challenges they address
- Different environments (dev/test/prod)
- Scalability requirements
- Security needs

17. What is virtualization, and what are the key components of a virtualization stack?

Virtualization
Virtualization is the process of creating virtual versions of servers, OS, storage, or networks on one physical machine.

Key components of virtualization stack
- Physical hardware (server)
- Host OS (sometimes)
- Hypervisor (VMware, VirtualBox, Hyper-V)
- Guest OS
- Virtual Machines (VMs)
- Virtual resources (CPU, RAM, storage, network)

Simple point
- Virtualization allows multiple virtual machines to run on one physical server.

18. How does executing code on the client differ from server-side execution?

Client-side execution
- Runs in user’s browser/device
- Example: JavaScript in browser
- Faster UI interaction
- Less secure for sensitive logic
- Depends on client device

Server-side execution
- Runs on server
- Example: Java, Python, Node.js on server
- Handles business logic and database
- More secure for sensitive operations
- Controlled by server environment

Simple point
- Client-side = browser/device work
- Server-side = backend/server work

19. What is Puppet, and how do Puppet master and Puppet agents interact?

Puppet
Puppet is a configuration management tool used to automate:
- Server setup
- Software installation
- System configuration
- Deployment tasks

Puppet Master and Agent interaction
- Puppet Master stores configuration rules
- Puppet Agent runs on client machines
- Agent requests configuration from Master
- Master sends configuration
- Agent applies configuration on the system
- Agent sends status/report back

Simple flow
- Agent → asks Master → gets config → applies config

Azure
